Rencana — Meta-Skill Graduasi

Auto-generate skill spesifik dari pola penugasan yang sudah dikerjakan (skill umum → skill spesifik)
Inspektorat II Komdigi · port orchestrator graduasi.py ke v7 · human-in-the-loop (promote manual)

1. Tujuan

Setelah beberapa penugasan dijalankan dengan skill umum (mis. reviu-umum) untuk objek yang sejenis, sistem bisa menyuling pola-nya (kriteria berulang, pola temuan, red-flag) menjadi draft skill spesifik baru — yang lalu direviu & di-promote auditor menjadi skill penuh. Ini menutup loop "umum → spesifik" pada taksonomi skill.

2. Yang sudah ada (modal)

Tim cowork sudah menulis orchestrator lengkap: knowledge/skills/graduasi-skill-spesifik/scripts/graduasi.py + SKILL.md + 3 reference. Sudah punya fungsi deterministik:

FungsiKeterangan
validate_inputscek penugasan (context.md + temuan.json + skill umum yang sama)
detect_domain_terms / detect_core_regulationsfrekuensi term & regulasi inti (anti-stopword ID)
suggest_skill_name / consolidate_kriteriausul nama skill + konsolidasi kriteria berulang
cluster_temuan (Jaccard)kelompokkan pola temuan → kandidat red-flag
write_skill_md / write_regulasi_md / write_checklist_redflagtulis draft ke skills/_draft/<nama>/

Kendala port: script meng-asumsikan layout audit-system-v4/: root/penugasan/<id>, root/skills, root/feedback/graduasi-log.json. Di v7, penugasan ada di folder data app (kode penugasan, bukan id sederhana) & skill di knowledge/skills. Promote harus memicu refresh registry v7.

4. Fase

FASE 1  Port script + bridge

  • Parametrasi graduasi.py: argumen --penugasan-dir, --skills-dir, --draft-dir, --log (default dari env v7: data_dir/penugasan, knowledge/skills, knowledge/skills/_draft). Salin ke folder app (mis. backend/app/scripts/graduasi.py) supaya tak menyentuh knowledge/ saat eksekusi — atau panggil in-place read-only.
  • Bridge tool run_graduasi(args) (mirip run_v6_script) untuk operasi deterministik: list kandidat, generate draft, promote, reject.
  • Endpoint baca: GET /graduasi/candidates (penugasan per skill umum, status SELESAI), GET /graduasi/drafts.

FASE 2  Agen + gate generate draft

  • Agen graduasi (atau mode PT/PM): load_skill("graduasi-skill-spesifik") → jalankan gate 0–4. Reuse pola gate engine (state graduasi-progress.json) bila perlu resume.
  • Output draft di knowledge/skills/_draft/<nama>/: SKILL.md (status DRAFT), references/01-regulasi.md, references/02-checklist-redflag.md + meta domain/kriteria.
  • Endpoint aksi: POST /graduasi/run {penugasan_ids, nama?} → draft.

FASE 3  Reviu & promote + UI

  • POST /graduasi/promote {nama} → pindah _draft/<nama> → knowledge/skills/<nama> + skills_registry.refresh() (skill langsung muncul di dropdown). POST /graduasi/reject {nama}.
  • UI: panel di tab Knowledge — daftar kandidat (grup per skill umum), tombol "Graduasikan", daftar draft + tombol "Reviu/Promote/Tolak", preview SKILL.md draft.
  • Audit trail: log graduasi (siapa, kapan, dari penugasan mana) di feedback store.

5. Tingkat keyakinan output (dari SKILL cowork)

Jumlah penugasanKualitas draft
1 (Minimum)skill kerangka — kriteria + 2-3 red-flag; wajib reviu berat
2–4 (Standar)kriteria stabil + pola temuan ter-cluster
≥5 (Penuh)checklist red-flag matang, layak jadi skill spesifik

6. Risiko & non-goals

  • Human gate wajib — draft TIDAK auto-aktif; promote selalu manual (kualitas + tanggung jawab profesional).
  • Penugasan v7 di DB+folder (kode, bukan id sederhana) → adaptasi find_penugasan ke layout v7.
  • Promote menulis ke knowledge/skills/ (di-track git) → auditor commit hasil promote; registry refresh agar langsung terpakai.
  • Bukan tujuan: auto-graduasi terjadwal; generate pipeline Python deterministik (skill hasil graduasi tetap criteria-driven dulu, bisa "naik kelas" lewat jalur lain).
  • Tidak mengarang regulasi/kriteria — semua disuling dari penugasan nyata + verifikasi auditor.

Disusun untuk persetujuan. Modal besar (script sudah ada) → fokus port + integrasi + UI. Urutan: Fase 1 (port) → 2 (generate) → 3 (promote+UI).
