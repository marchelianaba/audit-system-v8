1. Ringkasan & Keputusan

Anda menambahkan folder skills/ (22 entri dari sistem cowork lama). v7 saat ini hanya mengenal 2 skill. Dokumen ini merencanakan satu mesin skill generik (criteria-driven) sehingga skill pengawasan lain bisa dijalankan tanpa membangun pipeline Python baru per skill.

PendekatanHybrid. Bangun mesin generik dulu; skill yang sering dipakai "digraduasi" jadi pipeline deterministik kemudian (sesuai meta-skill graduasi-skill-spesifik).
Pilotaudit-kinerja. Skill naratif KKSA penuh (kondisi→sebab→akibat→rekomendasi) + survey pendahuluan. Cocok untuk memvalidasi reuse renderer KKP/LHP v7.
OutputKKSA v7 dulu. Pakai ulang append_temuan → render_kkp/render_lhp. Format asli (LHE/LHR-tabular/Memo) disusulkan di Fase C.

3. Lima celah integrasi harus ditutup

G1 — v7 hanya kenal 2 skill. Skill enum (models.py) + skema (schemas.py) + dropdown frontend hardcode reviu-rka-kl/reviu-pengadaan.
G2 — SKILL.md tak pernah dimuat. Agen AT/KT mengikuti prompts/anggota_tim.md & ketua_tim.md (berbentuk RKA/PBJ), bukan file skill. Prosedur kaya di skills/ = "dead weight" bagi v7.
G3 — Tak ada pipeline deterministik untuk skill lain (hanya RKA/PBJ punya digest_*/render_*). Sebagian besar skill bersifat prosedural; hanya evaluasi-sakip membawa scripts.
G4 — Path menunjuk audit-system-v4/... di 17 SKILL.md (mis. audit-system-v4/scripts/..., /checklists/...) — tidak resolve di layout v7.
G5 — Format output beda per jenis (PANDUAN.md): Audit/Eval-Kinerja = KKSA penuh; Reviu SAKIP/SPIP = tabel per-kriteria tanpa "sebab"; Eval-RB = tabel 4 dimensi; Konsultansi = memo. Renderer v7 sekarang berasumsi KKSA.

✓ Sudah beres lebih dulu: list_temuan_patterns kini folder-driven (semua 12 kategori pattern terbaca), dan duplikat RKA-05 1.md dihapus.

Dua jalur input hidup berdampingan:

  • Jalur pipeline (RKA/PBJ, sudah ada): upload TOR/RAB/KAK → digest V6 → _INGESTED.
  • Jalur criteria-driven (skill lain): auditor unggah kriteria/ + objek/ → tanpa digest, langsung READY → agen baca via read_pdf_page + ikuti prosedur SKILL.md.
Prinsip dipertahankan: V6 tetap read-only — skill baru hidup di path milik app (APP_SKILLS_PATH), bukan di backend/v6/. Tidak ada skema DB baru yang wajib (skill sudah disimpan sebagai String; cukup longgarkan validasi enum).
