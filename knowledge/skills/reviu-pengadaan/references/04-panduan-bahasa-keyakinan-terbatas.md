# Panduan Bahasa: Keyakinan Terbatas vs Audit Penuh

## Konsep Keyakinan Terbatas (Limited Assurance)

Reviu dokumen menghasilkan **keyakinan terbatas** — berbeda dari audit yang menghasilkan keyakinan memadai (reasonable assurance). Perbedaan ini harus tercermin dalam bahasa laporan.

---

## Template Kalimat Kesimpulan

### Jika Semua Aspek Terpenuhi (Laporan Bersih):
```
"Berdasarkan hasil reviu tidak terdapat hal-hal yang membuat kami yakin
bahwa [aspek yang direviu] atas [nama pekerjaan/objek] tidak terpenuhi."
```

**Contoh aktual dari laporan:**
> "Berdasarkan hasil reviu tidak terdapat hal-hal yang membuat kami yakin bahwa justifikasi kontrak pembayaran, kewajaran harga dan pemenuhan output pekerjaan atas Pekerjaan [...] tidak terpenuhi."

---

### Jika Ada Catatan Minor:
```
"Berdasarkan hasil reviu, terdapat hal yang perlu mendapat perhatian yaitu
[deskripsi catatan]. Kami merekomendasikan kepada [pihak yang bertanggung jawab]
untuk [tindakan korektif] paling lambat [tenggat waktu]."
```

---

### Jika Aspek Tidak Dapat Diverifikasi:
```
"Reviu tidak dapat memberikan keyakinan atas [aspek] dikarenakan
[dokumen/data] tidak tersedia pada saat reviu dilaksanakan.
Kami merekomendasikan agar [tindakan pengadaan dokumen/data]."
```

---

## Kata-Kata yang BOLEH vs TIDAK BOLEH

| ✅ BOLEH (Bahasa Reviu) | ❌ TIDAK BOLEH (Bahasa Audit) |
|------------------------|-------------------------------|
| "catatan reviu" | "temuan audit" |
| "perlu mendapat perhatian" | "penyimpangan" |
| "dapat diyakini" | "tidak terdapat penyimpangan" |
| "terdapat hal yang..." | "terdapat temuan berupa..." |
| "rekomendasi" | "rekomendasi perbaikan" (boleh jika perlu) |
| "keyakinan terbatas" | "simpulan audit" |
| "telah terpenuhi" | "patuh/tidak patuh" |

---

## Struktur Kalimat per Aspek

### Aspek Justifikasi:
```
"Berdasarkan [dasar hukum/peraturan], [nama pekerjaan] telah termuat dalam
[RKA-KL/DIPA/dokumen perencanaan]. Selain itu, [pihak] telah [tindakan
yang membuktikan keberlanjutan]. Sehingga justifikasi untuk [tindakan]
telah terpenuhi."
```

### Aspek Kewajaran Harga:
```
"Berdasarkan hasil reviu atas [dokumen harga] dan [referensi pembanding],
didapatkan [konsistensi/kewajaran/ketidaksesuaian] antara [X] dan [Y].
Sehingga [nama pekerjaan] dapat diyakini kewajaran harganya."
```

### Aspek Output/SLA:
```
"Dalam melakukan reviu atas hasil pekerjaan, [unit APIP] membandingkan
[SLA dalam Laporan Penyedia] dengan [Laporan Pembuktian SLA oleh instansi].
Hasil perbandingan menunjukkan [konsistensi/ketidaksesuaian] dengan SLA
[di atas/di bawah] [persentase]%."
```
