# Glossary — Istilah & Akronim Komdigi + Pengawasan Internal

**Tujuan halaman ini**: cheat sheet definisi istilah teknis yang dibaca agen ketika menemukan akronim di dokumen yang dianalisis. **Cegah agen mendefinisikan istilah dengan asumsi**.

**Catatan untuk agen**:
- Bila menemukan istilah yang TIDAK ADA di sini, JANGAN definisikan sendiri. Tulis temuan dengan istilah verbatim dari dokumen + kirim `submit_feedback` agar auditor menambahkan definisi.
- Istilah dengan tanda *(perlu verifikasi)* belum terkonfirmasi penuh dari dokumen sumber — agen wajib catat ini dan tidak boleh menyimpulkan definisi.

## A – C

| Istilah | Kepanjangan / Makna |
|---|---|
| **Adendum** | Perubahan resmi atas kontrak yang sudah ditandatangani, untuk penyesuaian harga, waktu, atau ruang lingkup. |
| **APIP** | Aparat Pengawasan Intern Pemerintah — payung hukum bagi Itjen K/L dan Inspektorat Daerah. |
| **Aptika** | Direktorat Jenderal Aplikasi Informatika (nomenklatur lama Kominfo); fungsinya kini terbagi ke Ditjen Ekosistem Digital + Ditjen Wasdig. |

| **BAHP** | Berita Acara Hasil Pemilihan — dokumen hasil evaluasi pemilihan penyedia. |
| **BAST** | Berita Acara Serah Terima — konfirmasi pekerjaan/barang diterima; dasar pembayaran. |
| **BMN** | Barang Milik Negara. |
| **BPK** | Badan Pemeriksa Keuangan — pemeriksa eksternal (UUD 1945). |
| **BPKP** | Badan Pengawasan Keuangan dan Pembangunan — auditor internal pemerintah. |
| **BUN** | Bendahara Umum Negara (BA 999.08 = Bagian Anggaran BUN tempat anggaran K/L tertentu, termasuk LPU). |
| **CaLK** | Catatan atas Laporan Keuangan. |
| **CBA** | Cost-Benefit Analysis. |
| **CHR** | Catatan Hasil Reviu — bentuk output reviu lebih singkat dari LHR. |
| **CNS** | Content Negative System (padanan CSE untuk konten tertentu di Wasdig — *perlu verifikasi*). |
| **CSE** | Content Security Engine — sistem pendukung pemutusan akses/takedown konten negatif di Ditjen Wasdig. |

## Cara Pakai untuk Agen

1. **Saat menemukan akronim di dokumen**: pertama, panggil `get_konteks("glossary")` (atau gunakan halaman ini) untuk cari definisi.
2. **Bila tidak ada di glossary**: gunakan istilah verbatim dari dokumen, jangan definisikan sendiri. Kirim `submit_feedback` dengan catatan "istilah baru: {istilah} — mohon ditambahkan".
3. **Bila ada tanda *(perlu verifikasi)***: jangan simpulkan definisi. Tulis catatan di KKP bahwa definisi masih perlu konfirmasi auditor.
4. **Untuk vendor**: bila muncul nama vendor yang tidak ada di tabel, cek apakah ada referensi di pattern terkait atau LHR sebelumnya — jangan ngarang profil vendor.
